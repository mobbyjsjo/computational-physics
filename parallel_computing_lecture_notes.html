第五章 并行计算入门

本章面向零基础学习者，详细解释并行计算的核心概念，并配有直观例题分析，帮助你快速理解“为什么要并行、如何并行、常见工具和注意事项”。

5.1 并行运算概念

并行计算的目标是把一个大任务拆分成多个小任务，同时执行，从而缩短总时间。

关键概念：
  • 串行计算：任务一个接一个做。
  • 并行计算：任务被拆成多个部分，同时做。
  • 加速比：并行后时间缩短的倍数。$S = T_{serial}/T_{parallel}$
  • 效率：$E = S / P$，其中 $P$ 是使用的处理器数。
例题： 如果单核执行需要 100 秒，4 核并行后耗时 30 秒，加速比 $S=100/30=3.33$，效率 $E=3.33/4=0.83$。

5.2 多线程并行 OpenMP 基本用法

OpenMP 是一种共享内存并行编程工具，适用于多核 CPU。

核心思想： 用编译指令告诉编译器“这段循环可以并行”。 
#pragma omp parallel for
for (int i = 0; i < N; i++) {
    a[i] = b[i] + c[i];
}
例题： 给定向量加法，使用 OpenMP 并行化循环可显著加速，适合 N 很大的情况。

5.3 多线程并行 OpenMP 注意问题

常见问题：
  • 竞态条件：多个线程同时写同一个变量，结果不确定。
  • 临界区：需要加锁保护共享资源。
  • 负载不均：任务分配不均导致部分线程空闲。
例题： 计算总和时，如果多个线程直接累加同一变量会出错，应使用 reduction。

5.4 多进程并行 MPI 基本概念

MPI 适用于分布式内存环境（多台计算机或集群）。

核心概念：
  • 进程：独立运行的程序实例。
  • 通信：进程之间通过消息传递交换数据。
  • Rank：每个进程的编号。
例题： 把大矩阵分给多个进程，每个进程处理一块，再汇总结果。

5.5 多进程并行 MPI 通信方法

常用通信：
  • 点对点通信：Send/Recv。
  • 集合通信：Broadcast、Reduce、Scatter、Gather。
例题： 多个进程计算局部求和，使用 MPI_Reduce 汇总得到全局结果。

5.6 使用加速硬件 GPU

GPU 适合大规模并行计算（例如矩阵运算、图像处理、深度学习）。

概念要点：
  • SIMT：单指令多线程，GPU 的运行模式。
  • 线程块：GPU 线程的组织方式。
  • 内存带宽：GPU 性能关键瓶颈。
例题： 用 GPU 加速矩阵乘法通常比 CPU 快数十倍，但需要关注内存传输开销。

5.7 SIMD 指令集优化

SIMD（Single Instruction Multiple Data）是一种 CPU 向量化技术，一条指令可同时处理多个数据。

概念解释：
  • 向量寄存器：一次存多个数据。
  • 向量化：循环自动转换成 SIMD 指令。
例题： 对一个数组执行加法，SIMD 可以一次处理 4 或 8 个元素，从而提升吞吐。

总结：并行计算的本质是“时间换空间”，通过更多计算资源换取更短时间。OpenMP 适合共享内存，MPI 适合集群，GPU 擅长大规模并行，SIMD 提升 CPU 单核吞吐。理解这些概念后，你就能根据任务规模选择合适的并行工具。
